# Onboarding by Role

## Developer Onboarding

1. Complete [Local Environment Setup](../tutorials/developer/setup-local-env.md)
2. Read [Business Context](./business/business-context.md)
3. Read [Architecture (arc42)](./architecture/arc42.md)
4. Review [Folder Structure](./backend/folder-structure.md)
5. Review [Service Layer](./laravel/service-layer.md)
6. Review [Test Strategy](../reference/testing/test-strategy.md)
7. Set up IDE with PHPStan integration
8. Build and run a small feature end-to-end

## DevOps Onboarding

1. Complete [Local Environment Setup](../tutorials/developer/setup-local-env.md)
2. Read [Infrastructure Diagram](../reference/cloud/infrastructure-diagram.md)
3. Review [CI/CD Pipeline](../reference/devops/ci-cd-pipeline.md)
4. Review [Deployment Strategy](../reference/devops/deployment-strategy.md)
5. Review [Monitoring Architecture](../reference/monitoring/monitoring-architecture.md)
6. Review [Runbook](../reference/operations/runbook.md)
7. Verify staging deployment
8. Set up local dev environment access

## QA Onboarding

1. Read [Business Context](./business/business-context.md)
2. Review [Test Strategy](../reference/testing/test-strategy.md)
3. Review [Test Cases Reference](../reference/testing/test-cases-reference.md)
4. Run existing test suite
5. Write and submit one test case
6. Review incident response process
