# Onboard a New Developer

1. **Access provisioning:**
   - GitHub repository access
   - AWS IAM user (read-only + ECS exec)
   - Database read-only access (staging)
   - Slack channel invites
2. **Environment setup:**
   - Follow `tutorials/developer/setup-local-env.md`
   - Verify project runs locally
   - Run test suite
3. **Knowledge transfer:**
   - Read `tutorials/developer/onboarding.md`
   - Read `reference/business/business-context.md`
   - Review `reference/architecture/arc42.md`
   - Review `reference/backend/folder-structure.md`
4. **First tasks:**
   - Fix a small bug with a mentor
   - Write a unit test
   - Submit first PR
5. **Check-in:**
   - Weekly 1:1 for first month
   - Retrospective after first sprint
