## Summary

Software Engineer assigned to the APMS project, responsible for end-to-end feature development, system maintenance, quality assurance, and technical documentation. Reports to the Engineering Lead and collaborates closely with the DevOps and QA functions.

---

## Responsibilities

### Software Development

- Develop new features for the APMS Laravel application based on business requirements and prioritized backlog
- Improve existing modules to enhance usability, maintainability, and performance
- Refactor legacy code while maintaining application stability and backward compatibility
- Implement clean, maintainable, well-structured code following project coding standards (PSR-12, PHPStan Level 5)
- Participate in technical design and solution architecture for new business requirements

### Project Management (Jira)

- Track all development tasks, bugs, and improvements in **Jira**
- Maintain updated task status (To Do → In Progress → In Review → Done)
- Participate in sprint planning and retrospective meetings
- Reference Jira issue keys in all commit messages and pull requests
- Log estimated vs actual time for tasks

### Quality Assurance & Testing

- Write and maintain PHPUnit tests (Unit + Feature) for all new code
- Perform functional testing before deployment
- Verify regression coverage after modifications
- Identify, document, and report software defects in Jira

### Feature Maintenance

- Maintain existing application features
- Fix bugs reported via Jira or identified during testing
- Ensure cross-feature compatibility after system changes
- Monitor application performance and stability

### Documentation

- Update technical documentation for newly developed features (`docs/` directory)
- Document significant implementation decisions (ADR format)
- Record bug fixes, feature improvements, and configuration changes
- Follow Diátaxis documentation standards (Tutorials / How-To / Reference / Explanation)

### Collaboration

- Communicate progress regularly via daily standup (Slack)
- Discuss technical challenges before implementing major changes
- Participate in pull request reviews for team members
- Coordinate feature implementation based on Jira backlog priorities

### Code Quality

- Follow Laravel best practices
- Follow PSR-12 coding standards
- Enforce PHPStan Level 5 compliance
- Use Laravel Pint for automated code style enforcement
- Perform self-review before submitting pull requests
